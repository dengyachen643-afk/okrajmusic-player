# 小O的日音播放器

一个安静、偏私人气质的日音抽歌与播放小站。它从一份歌单长成了一个可以随机抽歌、翻专辑墙、按氛围筛选、直接播放音频，并生成分享图的小网页。

线上地址：

https://dengyachen643-afk.github.io/okrajmusic-player/

## 现在能做什么

- 播放 70 首日音曲目，音频文件托管在 Cloudflare R2，网页部署在 GitHub Pages。
- 随机抽一首歌，自动载入播放器，并显示对应推荐语。
- 按标签筛选：城市、夜色、后劲、器乐 / 数摇、动画记忆、律动。
- 用专辑 shelf 浏览音乐：同一张专辑会聚在一起，桌面端交叠悬浮，移动端可左右滑动每一层架子。
- 点击专辑会立刻播放，并打开专辑详情。
- 播放器支持随机、顺序、单曲循环、上一首、下一首、播放列表、进度条拖动。
- 专辑播放完后会继续随机到全局下一首，避免单曲专辑一直重复。
- 抽歌结果和专辑详情都可以生成分享图，包含推荐语、歌名、歌手、专辑、封面、站点署名和二维码。
- 移动端做了专门适配：专辑墙滑动、播放列表纵向滚动、按钮尺寸、分享入口和播放器跳转都做了整理。

## 项目结构

- `public/index.html`：网站入口，主要 UI、数据、播放器逻辑、专辑墙和分享图生成都在这里。
- `music-download-list.md`：下载与校对用的歌曲清单。
- `range_server.py`：本地预览用的 Range 请求服务器，方便音频进度条拖动。
- `data/`：Apple Music、QQ 音乐、网易云音乐的链接候选和备份数据。
- `.github/workflows/deploy-pages.yml`：GitHub Pages 自动部署配置。

音频文件没有放进 Git 仓库。当前播放器会从 R2 公共地址读取 `track-01.mp3` 到 `track-70.mp3`。

## 本地预览

推荐使用项目里的 Range 服务器：

```bash
python range_server.py 8765 public
```

然后打开：

```text
http://127.0.0.1:8765/
```

如果用普通静态服务器，部分浏览器里 MP3 进度条拖动可能不稳定。

## 部署

当前部署方式是 GitHub Pages。把改动 push 到 GitHub `main` 分支后，GitHub Actions 会自动发布 `public` 目录。

音频文件独立放在 Cloudflare R2。以后新增歌曲时，需要同时更新页面里的歌曲数据、R2 音频文件和下载清单。

## 这一版做过的事

- 从外部播放器方案切换到本地音频播放，解决 Apple Music 只能试听的问题。
- 把音频从仓库迁移到 R2，减少 GitHub 仓库体积，也让后续加歌更清爽。
- 重做播放器 UI：播放模式、上一首 / 下一首、播放列表、进度条和移动端按钮。
- 新增专辑墙：按专辑聚合、标签筛选、悬浮交叠、详情弹窗、点击即播。
- 修复多轮移动端问题：专辑墙横向滑动、浮动手感、纵向页面滚动、播放列表横向露白。
- 优化播放体验：预加载下一首、减少切歌等待，修复拖动进度条后音量键导致卡顿的问题。
- 新增分享图：Canvas 生成 PNG，复用站内专辑封面，处理中文标点和英文歌名换行。
- 更新页脚署名：歌单、文案、前后端都写进了小小的唱片内页。
